# scenario.md

# 금융 상품 비교 추천 AI 에이전트 사용자 시나리오

## 1. 문서 목적

본 문서는 Phase 1 프로젝트인 **금융 상품 비교 추천 AI 에이전트**의 사용자 시나리오를 정의한다.

`PRD.md`에서 정의한 요구사항을 실제 사용자 흐름 관점으로 풀어내며, 이후 `flow.md`, `IA.md`, `api-spec.md`, `data-definition.md`, `ai-policy.md` 작성의 기준으로 사용한다.

---

## 2. 주요 사용자 유형

| 사용자 유형 | 설명 | 주요 니즈 |
|---|---|---|
| 금융 초보 사용자 | 금융상품 비교 경험이 많지 않은 사용자 | 쉬운 설명, 추천 이유, 주의사항 |
| 목적 기반 사용자 | 목돈 마련, 비상금, 전세자금 등 목적이 명확한 사용자 | 목적에 맞는 상품 후보 탐색 |
| 금리 비교 사용자 | 예금·적금 금리를 빠르게 비교하려는 사용자 | 금리, 기간, 우대조건 비교 |
| 대출 탐색 사용자 | 대출 상품 조건을 참고 수준에서 비교하려는 사용자 | 금리 범위, 금융회사, 주의사항 |

---

## 3. 사용자 목표

| 번호 | 사용자 목표 | 설명 |
|---|---|---|
| GOAL-001 | 내 조건에 맞는 금융상품 후보를 찾는다 | 나이, 자산, 목적, 기간 등을 기준으로 상품을 탐색한다. |
| GOAL-002 | 상품별 차이를 이해한다 | 금리, 가입 기간, 우대 조건, 금융회사 차이를 비교한다. |
| GOAL-003 | AI 추천 이유를 확인한다 | 왜 해당 상품이 추천되었는지 자연스러운 설명으로 확인한다. |
| GOAL-004 | 주의사항을 확인한다 | 우대금리 조건, 가입 제한, 정보 변동 가능성을 확인한다. |
| GOAL-005 | 최종 가입 전 공식 정보를 확인한다 | 서비스 결과를 참고하고, 실제 가입 전 금융회사 정보를 확인한다. |

## 4. 입력 방식 시나리오

### 4.1 시나리오 개요

| 항목 | 내용 |
|---|---|
| 시나리오명 | 폼 입력 기반 추천 요청 |
| 사용자 | 금융상품을 비교하려는 일반 사용자 |
| 목적 | 자유 대화 입력 없이 정해진 조건을 입력해 추천 결과를 확인 |
| 기대 결과 | 입력값 기반 추천 결과와 AI의 대화형 설명 확인 |

### 4.2 사용자 입력 예시

| 입력 항목 | 입력 방식 |
|---|---|
| 상품 유형 | 버튼 선택 |
| 나이 | 숫자 직접 입력 |
| 자산 규모 | 금액 직접 입력 |
| 금융 목적 | 버튼 선택 또는 드롭다운 |
| 가입 기간 | 6개월 / 12개월 / 24개월 / 36개월 중 선택 |
| 선호 금융권 | 체크박스 또는 드롭다운 |
| 위험 선호도 | 버튼 선택 |

### 4.3 정상 흐름

1. 사용자는 자유 대화 입력창이 아니라 조건 입력 폼을 확인한다.
2. 사용자는 상품 유형과 필수 조건을 입력한다.
3. 사용자는 `추천 받기` 버튼을 클릭한다.
4. 시스템은 입력값을 검증하고 추천 요청을 처리한다.
5. AI는 추천 결과 화면에서 상품 후보와 추천 사유를 대화형 설명으로 제공한다.
6. 사용자는 추천 결과와 주의사항을 확인한다.

### 4.4 기대 결과

| 결과 영역 | 기대 내용 |
|---|---|
| 입력 UX | 사용자가 정해진 폼 항목을 통해 조건을 입력한다. |
| AI 설명 | 추천 결과 화면에서 자연스러운 문장으로 추천 사유를 설명한다. |
| Phase 1 범위 | 실시간 자유 대화형 챗봇은 제외한다. |
| 향후 확장 | 이후 Phase에서 챗봇 인터페이스를 확장할 수 있다. |

## 5. 대표 시나리오 1: 적금 상품 추천

### 5.1 시나리오 개요

| 항목 | 내용 |
|---|---|
| 시나리오명 | 사회초년생의 목돈 마련 적금 추천 |
| 사용자 | 20대 금융 초보 사용자 |
| 목적 | 매월 저축을 통해 목돈을 마련하고 싶음 |
| 상품 유형 | 적금 |
| 핵심 입력 | 나이, 월 저축 가능 금액, 가입 기간, 목적 |
| 기대 결과 | 조건에 맞는 적금 상품 3~5개와 추천 사유 확인 |

### 5.2 사용자 입력 예시

| 입력 항목 | 예시 값 |
|---|---|
| 상품 유형 | 적금 |
| 나이 | 29세 |
| 자산 규모 | 월 50만 원 저축 가능 |
| 금융 목적 | 목돈 마련 |
| 가입 기간 | 12개월 |
| 선호 금융권 | 은행 |
| 위험 선호도 | 안정성 우선 |

### 5.3 정상 흐름

1. 사용자는 메인 화면에서 서비스 설명과 면책 문구를 확인한다.
2. 사용자는 상품 유형으로 `적금`을 선택한다.
3. 사용자는 나이, 월 저축 가능 금액, 금융 목적, 가입 기간을 입력한다.
4. 사용자는 `추천 받기` 버튼을 클릭한다.
5. 시스템은 입력값 누락 여부를 검증한다.
6. 시스템은 금융감독원 금융상품통합비교공시 API 또는 캐시 데이터를 조회한다.
7. 시스템은 사용자 조건에 맞는 적금 상품 후보를 필터링한다.
8. 시스템은 상품 후보와 사용자 조건을 OpenAI 추천 요청에 전달한다.
9. AI는 추천 요약, 상품별 추천 사유, 비교 포인트, 주의사항을 생성한다.
10. 사용자는 추천 결과 화면에서 상품 목록과 설명을 확인한다.
11. 사용자는 최종 가입 전 공식 정보를 확인해야 한다는 안내를 확인한다.

### 5.4 기대 결과

| 결과 영역 | 표시 내용 |
|---|---|
| 추천 요약 | 20대 사회초년생의 12개월 목돈 마련 목적에 적합한 적금 상품 방향 |
| 추천 상품 | 최대 3~5개 적금 상품 |
| 비교 정보 | 금융회사, 상품명, 기본금리, 최고우대금리, 가입기간 |
| 추천 사유 | 목적 적합성, 금리 수준, 조건 단순성 |
| 주의사항 | 우대금리 조건, 가입 가능 여부, 금리 변동 가능성 |
| 면책 문구 | 참고용 도구이며 금융상품 가입 권유가 아님 |

## 6. 대표 시나리오 2: 예금 상품 추천

### 6.1 시나리오 개요

| 항목 | 내용 |
|---|---|
| 시나리오명 | 여유자금 예치 목적의 예금 추천 |
| 사용자 | 안정적으로 자금을 예치하려는 사용자 |
| 목적 | 일정 금액을 정해진 기간 동안 예치 |
| 상품 유형 | 예금 |
| 핵심 입력 | 예치 금액, 가입 기간, 안정성 선호 |
| 기대 결과 | 금리와 조건에 적합한 예금 상품 비교 |

### 6.2 사용자 입력 예시

| 입력 항목 | 예시 값 |
|---|---|
| 상품 유형 | 예금 |
| 나이 | 35세 |
| 자산 규모 | 1,000만 원 |
| 금융 목적 | 여유자금 예치 |
| 가입 기간 | 12개월 |
| 선호 금융권 | 은행 |
| 위험 선호도 | 안정성 우선 |

### 6.3 정상 흐름

1. 사용자는 상품 유형으로 `예금`을 선택한다.
2. 사용자는 예치 금액과 가입 기간을 입력한다.
3. 사용자는 안정성을 우선하는 조건을 선택한다.
4. 시스템은 예금 상품 데이터를 조회한다.
5. 시스템은 가입 기간과 금융권 조건에 맞는 상품 후보를 필터링한다.
6. AI는 사용자의 목적과 상품 조건을 비교해 추천 결과를 생성한다.
7. 사용자는 추천 상품과 상품별 차이를 확인한다.

### 6.4 기대 결과

| 결과 영역 | 표시 내용 |
|---|---|
| 추천 요약 | 안정적인 예치를 원하는 사용자에게 적합한 예금 상품 방향 |
| 추천 상품 | 최대 3~5개 예금 상품 |
| 비교 정보 | 금융회사, 상품명, 금리, 가입기간, 가입방법 |
| 추천 사유 | 안정성, 금리 수준, 조건 명확성 |
| 주의사항 | 중도해지 시 금리, 우대조건, 예금자보호 여부 확인 안내 |

## 7. 대표 시나리오 3: 대출 상품 탐색

| 항목 | 내용 |

| 시나리오명 | 생활자금 목적의 대출 상품 탐색 |

| 목적 | 생활자금 마련 |
| 상품 유형 | 대출 |
| 핵심 입력 | 필요 금액, 목적, 기간 |
| 기대 결과 | 대출 상품 후보와 주의사항 확인 |

### 7.2 사용자 입력 예시

| 입력 항목 | 예시 값 |
|---|---|
| 상품 유형 | 대출 |
| 나이 | 40세 |
| 자산 규모 | 필요 대출금 2,000만 원 |
| 금융 목적 | 생활자금 |
| 가입 기간 | 36개월 |

4. 시스템은 조회 가능한 대출 상품 후보를 구성한다.

6. 사용자는 대출 상품의 기본 조건과 주의사항을 확인한다.
7. 사용자는 실제 대출 가능 여부는 금융회사 심사를 통해 확인해야 한다는 안내를 확인한다.

| 주의사항 | 실제 금리와 한도는 개인 신용도 및 금융회사 심사 결과에 따라 달라짐 |
| 금지 표현 | 승인 가능, 반드시 유리함, 최저금리 보장 등 |

## 8. 재입력/수정 시나리오

### 8.1 시나리오 개요

| 항목 | 내용 |
|---|---|
| 시나리오명 | 추천 결과 확인 후 조건 일부 수정 |
| 사용자 | 첫 추천 결과를 보고 조건을 조정하려는 사용자 |
| 목적 | 기존 입력값을 유지한 상태에서 일부 조건만 바꿔 새 추천 결과 확인 |
| 기대 결과 | 이전 결과가 새 추천 결과로 대체됨 |

### 8.2 사용자 입력 예시

| 입력 항목 | 기존 값 | 수정 값 |
|---|---|---|
| 상품 유형 | 적금 | 적금 |
| 자산 규모 | 월 50만 원 | 월 70만 원 |
| 가입 기간 | 12개월 | 24개월 |
| 위험 선호도 | 안정성 우선 | 금리 우선 |

### 8.3 정상 흐름

1. 사용자는 추천 결과를 확인한다.
2. 사용자는 `조건 수정` 또는 입력 폼 영역으로 돌아간다.
3. 시스템은 기존 입력값을 유지한 상태로 폼을 표시한다.
4. 사용자는 일부 조건만 수정한다.
5. 사용자는 다시 `추천 받기` 버튼을 클릭한다.
6. 시스템은 새 입력값을 검증하고 추천 요청을 처리한다.
7. 로딩 중에는 추천 요청 버튼을 비활성화해 중복 요청을 방지한다.
8. 시스템은 이전 추천 결과를 새 추천 결과로 대체한다.

### 8.4 기대 결과

| 결과 영역 | 기대 내용 |
|---|---|
| 입력 유지 | 기존 입력값이 유지되어 일부 조건만 수정할 수 있다. |
| 결과 대체 | 이전 추천 결과는 새 결과로 대체된다. |
| 중복 방지 | 로딩 중 중복 클릭이 방지된다. |
| 사용자 이해 | 사용자는 조건 변경에 따른 추천 차이를 확인할 수 있다. |

## 9. 로딩/응답 대기 시나리오

### 9.1 시나리오 개요

| 항목 | 내용 |
|---|---|
| 시나리오명 | 금융 API 조회 및 AI 추천 생성 대기 |
| 사용자 | 추천 요청 후 응답을 기다리는 사용자 |
| 목적 | 처리 중 상태를 명확히 확인하고 중복 요청을 방지 |
| 기대 결과 | 로딩 상태와 실패 시 재시도 안내 제공 |

### 9.2 사용자 입력 예시

| 입력 항목 | 예시 값 |
|---|---|
| 상품 유형 | 예금 |
| 나이 | 35세 |
| 자산 규모 | 1,000만 원 |
| 금융 목적 | 여유자금 예치 |
| 가입 기간 | 12개월 |

### 9.3 정상 흐름

1. 사용자는 조건 입력 후 `추천 받기` 버튼을 클릭한다.
2. 시스템은 추천 요청 버튼을 비활성화한다.
3. 금융 API 또는 캐시 조회 중에는 상품 정보를 불러오는 중이라는 로딩 메시지를 표시한다.
4. AI 추천 생성 중에는 추천 사유를 작성 중이라는 로딩 메시지를 표시한다.
5. 사용자가 버튼을 반복 클릭해도 중복 요청은 발생하지 않는다.
6. 요청이 성공하면 추천 결과 화면을 표시한다.
7. 요청이 실패하면 오류 안내와 다시 시도할 수 있는 동선을 제공한다.

### 9.4 기대 결과

| 결과 영역 | 기대 내용 |
|---|---|
| 로딩 표시 | 금융 API 조회와 AI 추천 생성 상태를 구분해 안내한다. |
| 버튼 상태 | 추천 요청 버튼이 비활성화되어 중복 클릭을 방지한다. |
| 오류 처리 | 실패 시 원인을 이해할 수 있는 안내를 제공한다. |
| 재시도 | 사용자가 입력값을 유지한 상태로 다시 시도할 수 있다. |

## 10. 예외 시나리오

### 10.1 필수 입력값 누락

| 항목 | 내용 |
|---|---|
| 상황 | 사용자가 필수 입력값을 입력하지 않고 추천을 요청함 |
| 예시 | 상품 유형 또는 금융 목적 미입력 |
| 시스템 처리 | 추천 요청을 중단하고 누락 항목 안내 |
| 사용자 메시지 | 필수 정보를 입력한 뒤 다시 추천을 요청해 주세요. |

### 10.2 금융상품 데이터 조회 실패

| 항목 | 내용 |
|---|---|
| 상황 | 금융감독원 API 호출 실패 또는 응답 오류 |
| 원인 | API 키 누락, 네트워크 오류, 외부 API 장애 |
| 시스템 처리 | 오류 메시지 표시, AI 추천 요청 중단 |
| 사용자 메시지 | 금융상품 정보를 불러오지 못했습니다. 잠시 후 다시 시도해 주세요. |

### 10.3 추천 가능한 상품 없음

| 항목 | 내용 |
|---|---|
| 상황 | 사용자 조건에 맞는 상품 후보가 없음 |
| 원인 | 조건이 과도하게 제한적이거나 데이터 부족 |
| 시스템 처리 | 결과 없음 안내 및 조건 완화 제안 |
| 사용자 메시지 | 현재 조건에 맞는 상품을 찾기 어렵습니다. 가입 기간이나 금융권 조건을 조정해 보세요. |

### 10.4 OpenAI API 호출 실패

| 항목 | 내용 |
|---|---|
| 상황 | 상품 후보는 조회되었으나 AI 추천 생성 실패 |
| 원인 | API 키 누락, 사용량 제한, 응답 실패 |
| 시스템 처리 | 상품 후보 목록은 표시하고 AI 설명 실패 안내 |
| 사용자 메시지 | 상품 목록은 확인할 수 있지만, AI 추천 설명을 생성하지 못했습니다. |

### 10.5 대출 상품 단정 표현 방지

| 항목 | 내용 |
|---|---|
| 사용자 행동 | 사용자가 대출 승인 가능성 또는 실제 적용 금리를 질문한다. |
| 시스템 응답 | 승인 여부와 실제 금리는 금융회사 심사 결과에 따라 달라진다고 안내한다. |
| 사용자 메시지 | 실제 대출 가능 여부와 적용 금리는 개인 신용도, 소득, 금융회사 심사 결과에 따라 달라집니다. |
| 정책 상세 | 금지 표현 상세 목록은 `ai-policy.md`에서 정의한다. |

## 11. AI 응답 시나리오

### 11.1 AI가 수행해야 하는 것

| 번호 | 수행 항목 | 설명 |
|---|---|---|
| AI-001 | 상품 후보 요약 | 조회된 상품 후보를 핵심 조건 중심으로 요약한다. |
| AI-002 | 사용자 조건 반영 | 나이, 목적, 금액, 기간을 고려해 설명한다. |
| AI-003 | 추천 사유 작성 | 상품별 추천 이유를 자연어로 작성한다. |
| AI-004 | 비교 포인트 작성 | 상품 간 주요 차이를 정리한다. |
| AI-005 | 주의사항 작성 | 우대조건, 금리 변동, 가입 가능성 확인사항을 안내한다. |
| AI-006 | 면책 기준 준수 | 금융 자문이나 가입 권유로 보이지 않게 표현한다. |

| 번호 | 금지 항목 | 설명 |
|---|---|---|

| BAN-002 | 수익 보장 | "무조건 이득입니다", "반드시 유리합니다" 금지 |
| BAN-003 | 대출 승인 단정 | "승인 가능성이 높습니다" 같은 표현 금지 |
| BAN-004 | 개인별 금리 단정 | 실제 적용 금리를 확정적으로 말하지 않음 |
| BAN-005 | 원천 데이터 없는 정보 생성 | 조회되지 않은 상품 조건을 임의로 생성하지 않음 |
