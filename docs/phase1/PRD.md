# PRD.md

# 금융 상품 비교 추천 AI 에이전트 PRD

## 1. 문서 목적

본 문서는 Phase 1 프로젝트인 **금융 상품 비교 추천 AI 에이전트**의 제품 요구사항을 정의한다.

Phase 1의 목표는 사용자가 폼으로 입력한 조건을 기반으로 금융감독원 금융상품통합비교공시 API 데이터를 조회하고, 조건에 맞는 예금·적금·대출 상품 후보를 비교한 뒤 AI가 추천 사유를 대화형 문장으로 설명하는 MVP를 구현하는 것이다.

Phase 1은 자유 대화형 챗봇이 아니라 **사용자 조건 입력 기반 추천 서비스**를 기본 UX로 한다. 추천 결과 화면에서는 AI가 비교 근거와 주의사항을 자연스러운 설명형 문장으로 제공한다.

## 3. 문제 정의

일반 사용자는 예금, 적금, 대출 상품을 비교할 때 다음과 같은 어려움을 겪는다.

1. 금융기관별 상품 조건이 분산되어 있어 비교가 어렵다.
2. 금리, 가입 조건, 우대 조건, 만기 조건 등 판단 요소가 많다.
3. 본인의 목적과 상황에 맞는 상품을 직접 선별하기 어렵다.
4. 검색 결과는 많지만 비교 기준이 명확하지 않다.

본 프로젝트는 이러한 문제를 해결하기 위해 사용자의 조건을 바탕으로 금융상품 데이터를 조회하고, AI가 비교 근거와 함께 추천 결과를 제공하는 MVP를 구현한다.

## 4. 서비스 목표

### 4.1 핵심 목표

- 사용자의 조건을 입력받아 적합한 금융상품 후보를 조회한다.
- 조회한 상품 데이터를 기반으로 AI가 비교·요약·추천한다.
- 추천 사유와 주의사항을 대화형 설명 문장으로 제공한다.
- 금융 자문이 아닌 참고용 탐색 도구임을 명확히 안내한다.

### 4.2 포트폴리오 목표

- 금융 도메인 데이터 사용 경험을 보여준다.
- RAG 기반 AI 서비스 설계 역량을 보여준다.
- 기획 문서에서 사용자 조건, 데이터, AI 응답 정책을 구조화한 역량을 보여준다.
- 단순 자유 대화형 챗봇이 아니라 폼 입력 기반 추천 흐름을 구현한다.

## 9. AI 응답 원칙

AI는 다음 원칙을 준수해야 한다.

1. 금융상품 가입을 직접 권유하지 않는다.
2. 특정 금융회사나 상품이 절대적으로 우수하다고 표현하지 않는다.
3. 사용자의 입력 조건과 조회한 상품 데이터에 기반해 설명한다.
4. 불확실한 정보를 단정하지 않는다.
5. 금리, 조건, 상품 내용은 변동될 수 있음을 안내한다.
6. 추천 결과는 참고용 정보임을 명시한다.
7. 사용자가 최종 가입 전 금융회사 공식 정보를 확인해야 한다고 안내한다.

## 12. API 개요

Phase 1에서는 다음 API 구성을 기본으로 한다.

| API | 설명 |
|---|---|
| `GET /health` | 서버 상태 확인 |
| `POST /api/phase1/recommendations` | 사용자 조건 기반 금융상품 추천 요청 |
| `GET /api/phase1/products` | 금융상품 데이터 조회 또는 캐시 데이터 확인 |

`POST /api/phase1/recommendations`는 Phase 1의 핵심 추천 API이며 다음 처리를 한 번에 수행한다.

1. 사용자 입력값 검증
2. 금융상품 데이터 조회 또는 캐시 조회
3. 조건 기반 상품 후보 필터링
4. OpenAI 기반 추천 사유 및 비교 요약 생성

별도 `POST /api/phase1/ai-summary` 엔드포인트는 Phase 1 API 목록에서 제외한다. AI 요약 분리 API는 Phase 2 이후 검토한다.

상세 API 명세는 `api-spec.md`에서 정의한다.

| SC-006 | 배포 가능 | Render Free 티어 환경에서 MVP를 실행할 수 있다. |
| SC-007 | 포트폴리오 설명 가능 | 프로젝트 목적, 구조, AI 사용 방식, 제약 조건을 설명할 수 있다. |

## 15. 주요 리스크

| 리스크 | 설명 | 대응 방향 |
|---|---|---|
| 금융 API 응답 구조 복잡성 | 상품 유형별 필드가 다를 수 있음 | MVP에서는 핵심 필드만 우선 사용 |
| API 호출 제한 | 외부 API 호출이 많아질 수 있음 | 캐시 또는 샘플 데이터 사용 검토 |
| AI 추천의 과도한 단정 | 금융 자문처럼 보일 위험 | `ai-policy.md`에서 금지 표현 정의 |
| Render Free 제약 | 슬립 모드, 리소스 제한 가능 | 초기 로딩 지연 안내 및 경량 구조 유지 |
| 상품 데이터 최신성 | 금리와 조건이 변경될 수 있음 | 조회 시점과 공식 확인 안내 표시 |
| 프로젝트 범위 증가 | 2주 내 구현 범위 초과 가능 | 예금/적금 우선 구현 및 대출 제한적 탐색 |
